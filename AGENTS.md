# Project Instructions

## General

- Do not create extraneous docuemnts like CHANGELOG.md, IMPLEMENTATION_SUMMARY.md, or other unecessary documentation
- Do not create example files with example code, write tests instead

## Documentation

- Keep the README file up-to-date
- Keep the README easy to read and as short as possible
- The README serves as the primary place for documentation, do not create other documentation files

## Tests

- Use the latest Swift Testing framework (e.g. @Test and #expect)
